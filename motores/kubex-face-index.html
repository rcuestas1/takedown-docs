KubexFaceIndex — Indice Facial Web-Crawled ​

Visao Geral ​

O KubexFaceIndex e o banco de dados proprietario de embeddings faciais extraidos de imagens encontradas na web por todos os motores do Kubex. Permite busca por similaridade facial em tempo real — o usuario faz upload de uma selfie e o sistema retorna todas as ocorrencias daquela face encontradas em sites monitorados.

Modelo de negocio: Thumbnail (150px) visivel gratuitamente; URLs de origem requerem plano Pro+ (unlock por credito).

Arquitetura ​

Stack ​

  • Vector DB: Milvus 2.4 (standalone, HNSW index, COSINE metric)
  • Embedding model: InsightFace AntelopeV2 (512D float32)
  • Metadata: PostgreSQL 16 (web_face_sources)
  • Thumbnails: Cloudflare R2 (bucket kubex-faces, presigned URLs 1h)
  • Worker: celery_face queue separada (Dockerfile.face, 3 replicas)
  • GPU: Habilitada por padrao (INSIGHTFACE_GPU=true), fallback automatico para CPU se CUDA indisponivel

Pipeline de Ingestao ​

Crawler/8/9/13/15/18/27 encontra imagem
  → face_index_ingest_from_url (Celery, fila celery_face)
  → download (httpx, max 10MB)
  → decode (OpenCV)
  → face_detection (InsightFace, det_thresh=0.5)
  → [0 faces] → descarta
  → [1+ faces, max 10] → para cada face:
      • filtro qualidade (det_score >= 0.5) + tamanho (>= 64px)
      • opt-out check (Milvus kubex_face_optouts, similarity > 0.90)
      • thumbnail_crop (face + 20% margem, resize 150px, JPEG 75%)
      • pHash (imagehash, 64-bit, para dedup)
      • dedup tiered:
          - > 0.95: mesma foto → adiciona source, nao duplica
          - 0.85-0.95: mesma pessoa → agrupa sob mesmo face_id
          - < 0.85: nova face → insere no Milvus
      • thumbnail → R2 (faces/{YYYY-MM}/{uuid}.jpg)
      • metadata → PostgreSQL (web_face_sources)

Milvus Collections ​

kubex_web_faces ​

  • Schema: face_id (PK), embedding (FLOAT_VECTOR 512), source_domain, site_category, first_seen, image_phash, quality_score
  • Index: HNSW (M=16, efConstruction=200, COSINE)
  • Partitions: tube_sites, leak_sites, escort_sites, forums, imageboards, social_media, general_web

kubex_face_optouts ​

  • Schema: optout_id (PK), embedding (FLOAT_VECTOR 512), email_hash, created_at
  • Index: HNSW (M=16, efConstruction=200, COSINE)

API Endpoints ​

Busca facial (usuario) ​

  • POST /v1/face-search/web/detect — Detecta faces na imagem, retorna thumbnails 240x240 para selecao
  • POST /v1/face-search/web — Upload selfie + face_index, busca no indice (requer consentimento + Pro+)
  • POST /v1/face-search/web/unified — Busca unificada (usa embedding armazenado ou selfie)
  • POST /v1/face-search/web/instant — Busca instantanea Milvus-only (freemium, aceita face_index)
  • POST /v1/face-search/web/unlock — Desbloquear URLs de um face_id

Opt-out (publico, LGPD — pagina /opt-out no rodape do site) ​

  • POST /v1/face-search/opt-out — Enviar selfie + documento de identidade + email + nome (rate: 3/hora)
  • GET /v1/face-search/opt-out/confirm?token=xxx — Confirmar via email

Admin ​

  • GET /v1/admin/face-index/stats — Estatisticas gerais
  • GET /v1/admin/face-index/health — Saude Milvus + collection
  • GET /v1/admin/face-index/top-domains — Top 20 dominios
  • POST /v1/admin/face-index/force-stats — Disparar stats manualmente
  • POST /v1/admin/face-index/force-dedup — Disparar dedup manualmente
  • GET /v1/admin/face-index/opt-outs/stats — Resumo de opt-outs
  • GET /v1/admin/face-index/opt-outs — Listar opt-outs (paginado)
  • DELETE /v1/admin/face-index/opt-outs/{id} — Rejeitar opt-out

R2 Storage (Thumbnails) ​

  • Bucket: kubex-faces (separado de kubex-evidence)
  • Key pattern: faces/{YYYY-MM}/{uuid}.jpg
  • Tamanho medio: ~8-12KB (150px, JPEG 75%)
  • Acesso: Presigned URLs com expiracao de 1h
  • Local (dev): /tmp/kubex-faces/

LGPD Compliance ​

  1. Opt-out gratuito com verificacao por email (impede abuso de remocao por terceiros)
  2. Indice opt-out separado (Milvus kubex_face_optouts) — consultado antes de cada ingestao
  3. Minimizacao: apenas embedding (2KB) + thumbnail crop (10KB); nunca a foto original
  4. Consentimento biometrico obrigatorio para buscar (FaceSearchConsent, Art. 11)
  5. Retencao: faces sem match ativo em 12 meses removidas automaticamente
  6. Anti-stalking: pHash da query armazenado (90 dias) para auditoria forense
  7. img_bytes deletados imediatamente apos extracao do embedding ([LGPD] tag no codigo)

Usuario ​

  • /dashboard/protecao — Upload selfie + resultados inline + opt-out link
  • /dashboard/perfil — Gestao de consentimento biometrico (ativar/revogar)

Recursos Adicionais ​

  • Batch insert: batch_insert_faces() para insercao em lote (reduz round-trips gRPC)
  • Rate limit configuravel: FACE_INDEX_INGEST_RATE_LIMIT=50/s (ajustavel por ambiente)
  • HNSW ef configuravel: MILVUS_SEARCH_EF=200 (maior valor = maior recall, maior latencia)
  • Integracao com bulk crawling pipeline: imagens de bulk crawls alimentam automaticamente o indice facial
  • Opt-out cache Redis: refresh a cada 1h via Celery Beat (face_index.refresh_optout_cache), elimina round-trip Milvus por face
