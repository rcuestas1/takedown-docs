Modulos Compartilhados ​

1. Retry com Backoff Exponencial (app/core/retry.py) ​

Funcao retry_with_backoff() usada por todos os motores:

python
result = await retry_with_backoff(
    lambda: client.get(url),
    max_retries=3,
    base_delay=1.0,      # delay = min(base * 2^attempt, max)
    max_delay=30.0,
    jitter=True,          # delay * random(0.5, 1.5)
    retry_on=(HTTPStatusError, TimeoutException),
    circuit=serpapi_circuit,  # integracao opcional
    operation_name="serpapi_search",
)
  • Backoff exponencial: delay = min(base_delay * 2^attempt, max_delay)
  • Jitter aleatorio: ±50% para evitar thundering herd
  • Retry-After: Respeita header RFC 7231
  • Circuit breaker: Verifica antes de cada tentativa
  • RetryExhausted: Excecao com last_exception e attempts

2. Rate Limit Header Parser (app/core/rate_limit_parser.py) ​

Parser centralizado para headers de rate limit por plataforma:

python
info = parse_rate_limit_headers(response.headers, "discord")
if should_pause(info):
    await asyncio.sleep(info.wait_seconds())
PlataformaHeaders Parseados
DiscordX-RateLimit-Remaining, X-RateLimit-Reset-After, Retry-After
RedditX-Ratelimit-Remaining, X-Ratelimit-Used, X-Ratelimit-Reset
Twitter/Xx-rate-limit-remaining, x-rate-limit-limit, x-rate-limit-reset
GenericoRetry-After (RFC 7231)

Pausa preventiva quando remaining < 5 (threshold configuravel).

3. Block Detector (app/core/block_detector.py) ​

Detector inteligente de bloqueios:

python
block = detect_block(status_code, headers, body, platform="discord")
if block.is_blocked:
    # block.block_type: "rate_limit", "captcha", "ban", "auth", "geo"
    # block.confidence: 0.0 - 1.0
    # block.retry_after: segundos (se disponivel)
    # block.should_retry: True se temporario

Deteccoes por camada:

  1. Status codes: 401, 403, 429, 444, 451, 503
  2. HTML patterns: captcha, Cloudflare challenge, access denied, rate limit
  3. Platform-specific:
    • Telegram: FLOOD_WAIT_X, CHANNEL_PRIVATE, USER_DEACTIVATED
    • Instagram: checkpoint_required, login_required
    • Discord: retry_after no JSON, codigos de erro
    • Reddit: erro 429, ban
    • Twitter: rate limit no JSON

4. Browser Fingerprint (app/engines/crawler/fingerprint.py) ​

Gera perfis completos de browser (Chrome, Firefox, Safari, Edge) com:

  • User-Agent realista (rotacao entre versoes 2025-2026)
  • Accept headers especificos do browser
  • Sec-CH-UA (Client Hints)
  • Accept-Language (pt-BR weighted)
  • Cache-Control randomizado

Proxy Rotation ​

Configuracao ​

INSTAGRAM_PROXY_URLS=["http://user:pass@ig-proxy1:8080"]

# Crawler
CRAWLER_PROXY_URLS=["http://user:pass@crawl-proxy1:8080"]

ProxyManager (Redis-backed) ​

O ProxyManager (app/engines/crawler/proxy_manager.py) gerencia scoring por proxy:

  • Metricas: success_rate, avg_latency, block_rate, last_used
  • Selecao: Melhor score primeiro (weighted scoring)
  • Rotacao: Round-robin via recency factor
  • Ban automatico: Skip proxies com >50% block rate

HTTP Client com Proxy ​

python
from app.core.http_client import get_proxied_client

async with get_proxied_client(proxy_url) as client:
    resp = await client.get(url)

Throttle Adaptativo ​

O DomainThrottle (app/engines/crawler/throttle.py) implementa o algoritmo AutoThrottle do Scrapy:

target_delay = latency / target_concurrency
new_delay = old_delay + (target_delay - old_delay) / 2.0
jittered_delay = delay * random(0.7, 1.3)
  • Redis-backed: Coordena entre workers Celery
  • Jitter ±30%: Dificulta deteccao de padrao
  • Per-domain: Cada dominio tem seu proprio delay
  • Min/Max: 2s - 30s (configuravel)

StealthClient (app/core/stealth_client.py) ​

Cliente HTTP unificado que aplica automaticamente todas as tecnicas anti-deteccao:

python
from app.core.stealth_client import StealthClient
from app.core.proxy_pool import get_proxy_pool

async with StealthClient(
    proxy_manager=get_proxy_pool("api"),
    platform="reddit",
    referer_context="navigation",
) as stealth:
    sr = await stealth.get(url)
    if not sr.is_blocked:
        data = sr.response.content

Features automaticas por request:

FeatureDescricao
Browser fingerprintHeaders completos (UA, Accept, Sec-CH-UA) via generate_browser_profile_ordered()
Header order shuffleRandomiza ordem dos headers (Host/UA ficam no topo)
Referer spoofingGera Referer plausivel (Google search, navegacao, imagem) com 15% de chance vazio
Sec-Fetch-Site dinamicoAjusta baseado no Referer (cross-site do Google, none sem Referer)
Cookie jar per domainPersiste cookies dentro da sessao
Proxy rotationUsa ProxyManager.get_best_proxy(), registra success/failure
Block detectionRoda detect_block() pos-response. Se bloqueado: rotaciona proxy e retenta (ate 2x)
CAPTCHA solvingSe captcha detectado e solver configurado, tenta resolver automaticamente

Retorno tipado:

python
@dataclass
class StealthResponse:
    response: httpx.Response
    is_blocked: bool = False
    block_type: str = "none"
    proxy_used: str | None = None
    captcha_solved: bool = False

Motores que usam StealthClient:

  • Reddit Monitor — download de imagens (API requests mantem UA bot por ToS)
  • SourceIntel — fetch de URLs candidatas
  • Instagram Monitor — download de imagens e fotos de perfil

CAPTCHA Solver (app/core/captcha_solver.py) ​

Resolucao automatica de CAPTCHAs com 3 providers suportados:

python
from app.core.captcha_solver import get_captcha_solver

solver = get_captcha_solver()  # None se nao configurado
if solver:
    solution = await solver.solve("recaptcha_v2", site_url, site_key)

Providers:

ProviderConfig
2CaptchaCAPTCHA_SOLVER_PROVIDER=2captcha
Anti-CaptchaCAPTCHA_SOLVER_PROVIDER=anticaptcha
CapSolverCAPTCHA_SOLVER_PROVIDER=capsolver

Tipos suportados: reCAPTCHA v2, reCAPTCHA v3, hCaptcha, Cloudflare Turnstile

Controle de custo:

  • Redis key kubex:captcha:cost:{YYYY-MM} tracking mensal
  • CAPTCHA_SOLVER_MONTHLY_BUDGET (default 50 USD) — para automaticamente quando excede
  • Graceful degradation: se nao configurado, StealthClient simplesmente reporta block

Extracao automatica de site_key:

  • solve_from_html(site_url, html) detecta tipo e extrai site_key do HTML automaticamente
  • Suporta: data-sitekey, grecaptcha.execute(), <div class="h-captcha">, <div class="cf-turnstile">

TLS Fingerprint Impersonation (app/core/tls_client.py) ​

Wrapper do curl_cffi para impersonar TLS fingerprint de browser real:

python
from app.core.tls_client import get_tls_client

client = get_tls_client(proxy="http://proxy:8080")
resp = await client.get(url)

Perfis suportados: Chrome 124-131, Edge 130-131, Safari 17.5, Firefox 133

Como funciona:

  • curl_cffi replica exatamente o JA3/JA4 fingerprint do browser selecionado
  • Cipher suites, extensoes TLS, ALPN, compression — identicos ao browser real
  • Perfil aleatorio a cada instancia via get_tls_client()
  • Fallback automatico para httpx se curl_cffi nao instalado

Config: STEALTH_TLS_IMPERSONATION=true para habilitar no StealthClient

Scan Adaptativo (app/core/adaptive_scheduler.py) ​

Ajusta frequencia de scan por motor/target baseado na taxa de bloqueio:

python
from app.core.adaptive_scheduler import should_scan_now, record_scan_result

if await should_scan_now("motor9", target_id):
    results = await scan(target)
    blocked = any(r.is_blocked for r in results)
    await record_scan_result("motor9", target_id, blocked)

Algoritmo (rolling window ~20 scans):

Block RateAcaoMultiplicador
> 50%Backoff agressivointervalo × 3.0
> 30%Backoff moderadointervalo × 2.0
> 10%Backoff leveintervalo × 1.5
< 5%Recovery gradualintervalo × 0.8

Limites: [ADAPTIVE_SCAN_MIN_INTERVAL_MINUTES, ADAPTIVE_SCAN_MAX_INTERVAL_MINUTES]

Redis key: kubex:adaptive:{motor}:{target_id} com TTL 48h

Funcoes auxiliares:

  • get_current_interval(motor, target_id) — intervalo atual em minutos (dashboard)
  • get_stats(motor, target_id) — scan_count, block_count, block_rate, interval, last_scan_at
