Status: Ativo Queue: celery_face (face-worker) Dependencias: scipy, Pillow, numpy, InsightFace (face-crop), onnxruntime (modelos ONNX) Mock mode: Retorna is_fake=False quando scipy nao esta instalado Gating: Pro/Premium/Founder

Arquitetura ​

Pipeline multi-camada com face-crop + modelos treinados:

Imagem → Face-Crop (InsightFace) → [6 camadas] → Score Final
                                     ├── DCT (espectral)
                                     ├── ELA (compressao)
                                     ├── Noise (ruido GAN)
                                     ├── Patches (coerencia local)
                                     ├── Boundary (zona de transicao face/fundo)
                                     ├── ONNX Model 1 (EfficientNet-B4 DeepfakeBench)
                                     └── ONNX Model 2 (EfficientNet-B7 DFDC winner)

Boundary Analysis (v3) ​

Nova camada inspirada em SBI (Self-Blended Images, CVPR 2022) e Face X-Ray (CVPR 2020). Analisa a zona de transicao ao redor da borda facial detectada por InsightFace:

  1. Gradiente de borda: Descontinuidade de gradiente na fronteira face/fundo
  2. Inconsistencia de ruido: Diferenca estatistica de ruido entre interior e exterior da face
  3. Suavizacao artificial: Detecta gaussian blur/feather usado no blending de face swaps

Requer face_info (bbox + orig_array) do crop_face_with_info(). Se nenhum rosto detectado, score = 0.

Face-Crop ​

Antes de analisar, o motor cropa a face usando InsightFace AntelopeV2 (mesmo modelo do FaceID). Reusa o face-worker existente (Dockerfile.face).

  • Se nenhum rosto detectado: analisa imagem completa
  • Se multiplos rostos: usa o maior
  • Margem de 20% ao redor do rosto
  • crop_face_with_info() retorna crop + bbox/landmarks para boundary analysis

Codigo: app/engines/face_crop.py

Modelos ONNX ​

Modelo 1: EfficientNet-B4 (DeepfakeBench) ​

  • Fonte: DeepfakeBench (effnb4_best.pth, 67.7 MB)
  • Arquitetura: EfficientNet-B4 treinado no FaceForensics++ (c23)
  • AUC: 93.89% within-domain, 77.18% cross-domain
  • Input: 256x256, normalizado com mean=[0.5,0.5,0.5], std=[0.5,0.5,0.5]
  • Output: 2 classes [real, fake] → softmax → prob_fake
  • Licenca: CC BY-NC 4.0
  • Localizacao: models/deepfake_efficientnet.onnx

Modelo 2: EfficientNet-B7 (DFDC Winner) — v3 ​

  • Fonte: selimsef/dfdc_deepfake_challenge (Kaggle DFDC Champion, 859 stars)
  • Arquitetura: EfficientNet-B7 (timm tf_efficientnet_b7_ns) com head Linear(2560, 1)
  • Dataset: DFDC (8 metodos de face swap, 960 sujeitos — mais diverso que FF++)
  • Input: 380x380, normalizado com ImageNet mean/std
  • Output: 1 logit → sigmoid → prob_fake
  • Licenca: MIT
  • Localizacao: models/dfdc_efficientnet_b7.onnx

Boosts de Confianca ​

Alem da media ponderada, o ensemble aplica boosts em casos especificos:

CondicaoBoostRazao
Ambos ONNX concordam (score > 0.5)+min(score)*10%Concordancia forte de 2 modelos independentes
Boundary score > 0.5+boundary*12%Forte sinal de face swap
3+ heuristicas >= 0.3 (sem ONNX)*1.3xMultiplas evidencias convergem

Compensacao JPEG ​

Imagens muito comprimidas (quality < 75) produzem artefatos que mimetizam sinais de deepfake nas heuristicas. O motor aplica penalizacao proporcional a qualidade JPEG estimada nos scores de DCT, ELA e Noise.

Os modelos ONNX NAO sao afetados pela penalizacao (ja sao robustos a compressao).

Fluxo de Execucao ​

  1. API recebe imagem → valida tipo/tamanho
  2. Envia para celery_face queue (task deepfake.analyze)
  3. Face-worker recebe → face-crop com InsightFace (retorna crop + face_info)
  4. Roda 4 heuristicas CPU (DCT, ELA, Noise, Patches)
  5. Roda boundary analysis (se face detectada, usa bbox da face_info)
  6. Roda modelos ONNX (EfficientNet-B4 + B7, se disponíveis)
  7. Calcula score ensemble ponderado → retorna resultado
  8. Fallback: se Celery falha, roda localmente sem face-crop/ONNX

Reaproveitamento de Motores ​

  • FaceID (Face Recognition): InsightFace para face-crop + bbox/landmarks
  • ShieldAI (NCII): face-worker container (onnxruntime, InsightFace ja instalados)
  • Queue celery_face: worker dedicado ja configurado

Seguranca e LGPD ​

  • LGPD: del img_bytes + del img_bytes_b64 imediatamente apos processamento em tasks e endpoints
  • Memory: face_info["orig_array"] removido com .pop() apos boundary analysis
  • Memory: todas as imagens PIL fechadas com .close() apos uso (previne leak de handles)
  • Memory: del img_bgr apos deteccao de face (libera copia BGR)
  • Path traversal: modelos ONNX validados para estar dentro de models/ antes de carregar
  • Race condition: flags _checked setadas APOS try/except (retry em caso de erro transiente)
  • Download: verificacao de tamanho do checkpoint, weights_only=True tentado antes de False
  • Fallback: se Celery falha, analise local sem face-crop/ONNX (graceful degradation)
