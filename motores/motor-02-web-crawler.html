Motor 02 — Crawler (Web Crawler com Scrapling) ​

O Crawler e o principal diferencial tecnico do Kubex. Alcanca sites, foruns e canais que o Google nao indexa. Utiliza o framework Scrapling como camada de scraping com anti-deteccao, substituindo httpx + Scrapy.

Arquivo principal: app/engines/crawler/service.pyCelery Queue: celery (default) Dependencias: scrapling>=0.2.9, httpx, Pyrogram

Configuracao ​

bash
# .env
CRAWLER_PROXY_URLS=[]              # Proxies para rotacao
CRAWLER_RETRY_COUNT=3              # Retentativas
CRAWLER_RETRY_DELAY=2.0            # Delay entre retentativas (s)
CRAWLER_MIN_THROTTLE_DELAY=2.0     # Throttle minimo (s)
CRAWLER_MAX_THROTTLE_DELAY=30.0    # Throttle maximo (s)
CRAWLER_FETCH_TIMEOUT_MS=30000     # Timeout de fetch (ms)
CRAWLER_MAX_IMAGES_PER_SCAN=50     # Max imagens por scan
CRAWLER_ENABLE_CLOUDFLARE_BYPASS=false  # Requer Camoufox
CRAWLER_PAGINATION_MAX_DEPTH=3     # Profundidade de paginacao
CRAWLER_DISABLE_RESOURCES=true     # Desabilitar CSS/JS/fontes
GALLERY_DL_ENABLED=true            # Layer 2 (gallery-dl)
GALLERY_DL_TIMEOUT=30.0            # Subprocess timeout (s)
YTDLP_ENABLED=true                 # Layer 3 (yt-dlp)
YTDLP_TIMEOUT=15.0                 # extract_info timeout (s)
KUBEXBOT_ENABLED=true              # User-Agent personalizado
KUBEXBOT_RESPECT_ROBOTS_TXT=true   # Verificar robots.txt

Fluxo ​

Arquitetura Geral ​

Armazena SOMENTE: pHash, face embeddings, URL de origem, timestamp, metadados do site.

Fetchers Disponiveis ​

O Scrapling oferece diferentes fetchers para diferentes cenarios:

FetcherBackendQuando usarVelocidade
AsyncFetchercurl_cffi (async)Sites simples sem anti-bot~1-3s
DynamicFetcherPlaywright (async)Sites com JS pesado / lazy load~3-5s
StealthyFetcherCamoufox (async)Sites com Cloudflare/anti-bot~3-8s

Graceful degradation: se o fetcher solicitado nao estiver disponivel, faz fallback automatico para o proximo nivel.

Sistema de 3 Camadas de Extracao ​

CamadaArquivoDescricao
Layer 1app/engines/crawler/rippers/*.pyCSS selectors do site, in-process
Layer 2app/engines/crawler/gallery_dl_extractor.pysubprocess gallery-dl --dump-json (GPL-safe)
Layer 3app/engines/crawler/ytdlp_extractor.pyyt_dlp.YoutubeDL.extract_info(download=False)
Fallbackapp/engines/crawler/link_extractor.pyRegex + CSS generico

Sub-componentes (Redis-backed) ​

ModuloArquivoFuncao
Content Cachecrawler/cache.pySHA-256 do HTML, TTL 24h — evita reprocessar paginas iguais
URL Dedupcrawler/dedup.pyRedis SET por target, TTL 7 dias — evita revisitar URLs
Throttle Adaptativocrawler/throttle.pyAutoThrottle cross-worker via Redis SET NX PX
Estatisticascrawler/stats.pyMetricas por alvo + globais (Redis HASH)
Link Extractorcrawler/link_extractor.pyimg[src], lazy-load, CSS background, paginacao heuristica
URL Skip Cachecrawler/url_skip_cache.pyPula URLs que falharam recentemente (TTL 7d)
Proxy Managercrawler/proxy_manager.pyScoring e rotacao por proxy (Redis-backed)

Para detalhes completos da arquitetura modular, veja Arquitetura do Crawler.

Estrutura de Diretorio ​

app/engines/
  +-- crawler/
      +-- __init__.py
      +-- service.py          # ScraplingCrawlerService (fetch_html + search + search_paginated)
      +-- cache.py            # ContentCache — Redis SHA-256 dedup de paginas (24h TTL)
      +-- dedup.py            # URLDeduplicator — Redis SET de URLs visitadas (7d TTL)
      +-- throttle.py         # DomainThrottle — Redis SET NX PX (cross-worker, AutoThrottle)
      +-- stats.py            # CrawlerStats — metricas por alvo + globais (Redis HASH)
      +-- link_extractor.py   # extract_image_urls() + extract_links() + detect_pagination()
      +-- rippers/            # CSS selectors por site (Layer 1)
      +-- gallery_dl_extractor.py  # gallery-dl subprocess (Layer 2)
      +-- ytdlp_extractor.py       # yt-dlp extract_info (Layer 3)

Endpoints ​

EndpointMetodoDescricao
/api/v1/proxy/imageGETProxy de imagem on-demand (SSRF guard + sanitize + 400px)

Rate limit proxy: 60 req/min por IP. Codigos: 200 (ok), 400 (SSRF block), 413 (>10MB), 429 (rate limit), 503 (timeout).

Seguranca & LGPD ​

  • Nao armazena imagens — processadas em memoria e descartadas (del img_bytes)
  • _is_safe_url() bloqueia RFC-1918, loopback, IMDS, non-HTTP(S) (SSRF guard)
  • Download streaming: httpx.AsyncClient.stream() + 64KB chunks + abort em 10MB
  • sanitize_image(): valida formato, remove EXIF, redimensiona para max 400px
  • Evidence thumbnail com TTL LGPD 90 dias (cleanup automatico via Celery Beat)
  • PII nunca em logs — apenas IDs de sessao
  • KubexBot User-Agent identificado + respeita robots.txt (configuravel)

Testes ​

  • Unit tests: tests/engines/test_crawler.py (SSRF guard, rate limiting)
  • Cobertura: 70%
  • Gaps conhecidos: Testes com sites dinamicos (JS rendering)

Relacionamentos ​

  • Depende de: Redis (cache, dedup, throttle, stats), Scrapling (fetchers), Motor 3 (pHash)
  • Consumido por: Motor 1 (Lens — fallback quando < 5 resultados), Motor 8 (Orchestrator)
  • Conflita com: Motor 1 (ambos fazem busca de imagens web)
