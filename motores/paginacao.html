Paginacao dos Motores de Busca ​

Paginacao automatica dos motores para maximizar a cobertura de resultados. Para a especificacao completa de cada motor, veja Motores de Busca. Para resiliencia e anti-bloqueio, veja Resiliencia dos Motores.

Lens — Busca por Imagem (SerpAPI) ​

Como funciona ​

O SerpAPI retorna resultados de busca reversa de imagem com paginacao via parametro start (offset). O motor percorre automaticamente multiplas paginas para obter mais resultados alem da primeira.

Pagina 1: start=0  (resultados 1-20)
Pagina 2: start=20 (resultados 21-40)
Pagina 3: start=40 (resultados 41-60)

Criterios de parada ​

  • Atingiu MOTOR1_SERPAPI_MAX_PAGES paginas
  • Pagina retornou 0 resultados
  • Campo serpapi_pagination.next ausente na resposta
  • Quota SerpAPI esgotada
  • Rate limit (HTTP 429) ou falha de circuit breaker

Variaveis de ambiente ​

VariavelPadraoDescricao
MOTOR1_SERPAPI_MAX_PAGES3Maximo de paginas por busca
MOTOR1_SERPAPI_RESULTS_PER_PAGE20Resultados por pagina

Impacto na quota ​

Cada pagina consome 1 chamada da quota SerpAPI. Com 3 paginas, uma busca consome ate 3 creditos.

Crawler — Web Crawler (Scrapling) ​

Como funciona ​

O crawler detecta links de paginacao na pagina HTML usando heuristica de parametros de URL:

Detecta: ?page=1 → ?page=2 → ?page=3
Detecta: ?p=1, ?pag=1, ?pagina=1, /page/1

Algoritmo _find_next_page_url() ​

  1. Extrai todos os links da pagina atual
  2. Identifica o parametro de paginacao atual (page, p, pag, pagina)
  3. Calcula o numero da proxima pagina (N+1)
  4. Procura nos links extraidos um que tenha o parametro com valor N+1
  5. Se encontrar, segue para essa URL; senao, para

Variaveis de ambiente ​

VariavelPadraoDescricao
CRAWLER_PAGINATION_MAX_DEPTH3Maximo de paginas por alvo
CRAWLER_MAX_IMAGES_PER_SCAN50Maximo de imagens por varredura

Resiliencia ​

  • Cache de resposta: Pula paginas cujo HTML nao mudou (hash comparison)
  • Throttling por dominio: Delay adaptativo entre requests (Redis-backed)
  • SSRF guard: Valida URLs antes de fetch (bloqueia RFC-1918, loopback, IMDS)

Fingerprint — Telegram Monitor ​

Como funciona ​

Usa o Pyrogram MTProto com paginacao via offset_id. Cada batch busca um bloco de mensagens, e o offset_id da mensagem mais antiga do batch anterior serve como cursor para o proximo.

Batch 1: offset_id=0 (50 msgs mais recentes)
Batch 2: offset_id=msg_id_mais_antigo_do_batch_1 (50 msgs anteriores)
Batch 3: offset_id=msg_id_mais_antigo_do_batch_2 (50 msgs anteriores)

Metodo fetch_recent_images_paginated() ​

python
await monitor.fetch_recent_images_paginated(
    channel_username="@canal_alvo",
    max_batches=3,   # opcional, padrao: MOTOR3_PAGINATION_MAX_BATCHES
    batch_size=50,    # opcional, padrao: MOTOR3_BATCH_SIZE
)

Criterios de parada ​

  • Atingiu MOTOR3_PAGINATION_MAX_BATCHES batches
  • Batch retornou menos mensagens que MOTOR3_BATCH_SIZE (fim do historico)

Variaveis de ambiente ​

VariavelPadraoDescricao
MOTOR3_PAGINATION_MAX_BATCHES3Maximo de batches por canal
MOTOR3_BATCH_SIZE50Mensagens por batch

BrandWatch — Busca por Nome/Marca ​

Como funciona ​

O BrandWatch tem tres fontes de dados, cada uma com sua propria paginacao:

SerpAPI Texto (Google Web Search) ​

Paginacao via parametro start (offset), identico ao Lens:

Pagina 1: start=0  (20 resultados organicos)
Pagina 2: start=20 (20 resultados organicos)
Pagina 3: start=40 (20 resultados organicos)

SerpAPI Site-Scoped (operador site:) ​

Mesma paginacao offset-based, aplicada a buscas site:dominio.com:

Pagina 1: site:instagram.com "Fulana" start=0  (10 resultados)
Pagina 2: site:instagram.com "Fulana" start=10 (10 resultados)

Template Scraping (forums, sites especificos) ​

Segue links de paginacao no HTML, mesma heuristica do Crawler:

https://forum.com/search?q=fulana&page=1 → page=2 → page=3

Variaveis de ambiente ​

VariavelPadraoDescricao
MOTOR4_SERPAPI_MAX_PAGES3Paginas SerpAPI (texto e site-scoped)
MOTOR4_TEMPLATE_MAX_PAGES3Paginas de template scraping

Discord Monitor — Discord Monitor ​

Como funciona ​

Usa a REST API v10 do Discord com paginacao via cursor after (message ID). Cada pagina busca ate 100 mensagens (limite da API), e o ID mais recente serve como cursor para a proxima pagina.

Pagina 1: GET /channels/{id}/messages?limit=100
Pagina 2: GET /channels/{id}/messages?limit=100&after={newest_id_pagina_1}
Pagina 3: GET /channels/{id}/messages?limit=100&after={newest_id_pagina_2}

Metodo fetch_recent_images_paginated() ​

python
images, newest_id = await monitor.fetch_recent_images_paginated(
    channel_id=123456789,
    after_message_id=last_known_id,  # cursor incremental
    max_pages=3,    # opcional, padrao: MOTOR9_PAGINATION_MAX_PAGES
    page_size=100,  # opcional, padrao: MOTOR9_PAGE_SIZE (max: 100)
)

Criterios de parada ​

  • Atingiu MOTOR9_PAGINATION_MAX_PAGES paginas
  • Pagina retornou menos mensagens que page_size (fim das mensagens)
  • HTTP 403 (sem permissao) ou HTTP 429 (rate limit)
  • Circuit breaker aberto

Variaveis de ambiente ​

VariavelPadraoDescricao
MOTOR9_PAGINATION_MAX_PAGES3Maximo de paginas por canal
MOTOR9_PAGE_SIZE100Mensagens por pagina (max API: 100)

Motores com Paginacao Nativa (Redes Sociais) ​

Os motores de redes sociais usam paginacao nativa das APIs das plataformas:

Reddit Monitor — Reddit ​

  • Tipo: Cursor-based (after token)
  • Retorno: (images, next_after) — o caller armazena next_after no DB para scans incrementais
  • Limite: ate 100 posts por request

X Monitor — Twitter/X ​

  • Tipo: Time-based (since_id)
  • Retorno: (images, newest_tweet_id) — armazena para busca incremental
  • Limite: 10-100 tweets por request

Instagram Monitor — Instagram ​

  • Tipo: Incremental (since_shortcode)
  • Retorno: (images, newest_shortcode) — armazena para busca incremental
  • Limite: ate 20 posts por request

Diagrama de Fluxo Geral ​

Configuracao via .env ​

Adicione ao seu .env para customizar a profundidade de paginacao:

env
# Lens: Busca por imagem (SerpAPI)
MOTOR1_SERPAPI_MAX_PAGES=3
MOTOR1_SERPAPI_RESULTS_PER_PAGE=20

# Crawler: Web Crawler
CRAWLER_PAGINATION_MAX_DEPTH=3

# Fingerprint: Telegram
MOTOR3_PAGINATION_MAX_BATCHES=3
MOTOR3_BATCH_SIZE=50

# BrandWatch: Busca por nome
MOTOR4_SERPAPI_MAX_PAGES=3
MOTOR4_TEMPLATE_MAX_PAGES=3

# Discord Monitor: Discord
MOTOR9_PAGINATION_MAX_PAGES=3
MOTOR9_PAGE_SIZE=100

Recomendacao: Em producao, mantenha os valores padrao (3 paginas) para balancear cobertura e custo de API. Aumente apenas para investigacoes especificas via trigger manual no painel admin.
