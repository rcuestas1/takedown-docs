BrandWatch — Monitoramento de Nome/Marca ​

Disponivel para: Essencial, Fundador, Pro e Premium. Implementado em: 2026-03-02. Arquivos: app/engines/name_variation.py, app/engines/name_search_engine.py, app/engines/text_normalizer.py, app/api/v1/name_search.py.

Problema ​

Criadores digitais tem seu nome/marca usados por piratas em sites, foruns e Telegram para vender conteudo ilegal sem usar nenhuma imagem da vítima. A busca reversa de imagem (Lens+Crawler) nao cobre esse caso. O BrandWatch resolve: o criador digita seu nome e o sistema busca automaticamente em multiplas fontes, incluindo variacoes usadas para escapar de buscas simples ("Ful@no", "ful.ano", etc.).

Arquitetura ​

NameVariationGenerator ​

Gera variações do nome para maximizar cobertura de busca:

Camada 1 — Leetspeak:

a→@,4  e→3  i→!,1  o→0  s→$,5  l→1  b→6  t→+

Limite: 15 combinacoes (previne explosao combinatoria).

Camada 2 — Separadores (usernames):

  • fulana → ful.ana, ful-ana, ful_ana
  • Nome composto: "Ana Paula" → ana.paula, anapaula, _anapaula

Camada 3 — Fonetica (jellyfish):

  • ph↔f, k↔c/qu, y↔i, th↔t, x↔s/z
  • Duplicacao de consoante: "Felipe" → "Fellipe", "Phelipe"

2 formas de saída:

  • text_variants — com caracteres especiais, para APIs de busca: ["Fulana", "Ful@na"]
  • username_variants — so alfanumerico + ./-/_, para crawling de perfis: ["fulana", "ful.ana"]

Perfis Legítimos (Allowlist) ​

O criador cadastra seus proprios perfis (POST /profile-allowlist). O BrandWatch filtra esses URLs dos resultados, eliminando falsos positivos.

python
# app/models/user_profile.py -> tabela user_profiles
# Campos: user_id, platform, url, username
# Constraint: UNIQUE(user_id, url)

# Antes de salvar SearchResult:
own_urls = {_normalize_url(p.url) for p in user_profiles}
results = [r for r in raw_results if _normalize_url(r["found_url"]) not in own_urls]

CrawlTarget (alvos de crawling) ​

python
# app/models/crawl_target.py -> tabela crawl_targets
# Campos:
#   name, platform_type ("web"|"forum"|"social"|"telegram")
#   scrape_method ("google_web"|"bing_web"|"stealthy"|"dynamic"|"basic"|"telegram")
#   search_url_template  -- "https://site.com/search?q={query}" (NULL para APIs)
#   is_global BOOLEAN    -- TRUE = admin criou, FALSE = usuario criou
#   user_id              -- NULL se global
#   is_active BOOLEAN

Admins criam alvos globais (/admin/targets). Usuarios Pro/Premium podem adicionar alvos pessoais (POST /crawl-targets).

Anti-Ofuscacao (text_normalizer.py) ​

Pipeline de normalizacao de texto para detectar nomes ofuscados. Todas as operacoes O(n).

5 estagios de normalizacao:

EstagioEntradaSaidaExemplo
1. Braille reversal⠁⠝⠁anaUnicode U+2801..U+283A → ASCII
2. HomoglifosАnа (cirilico)AnaCirilico, grego, math, fullwidth → ASCII
3. Leet reversalFul4n4Fulana4→a, 3→e, 1→i, 0→o, $→s
4. Symbol-initial&arolCarol&→C, %→G, #→B, @→B
5. Cleanup Ana AnaControl chars, multi-spaces

Homoglifos suportados:

  • Cirilico (26 chars): а→a, е→e, о→o, р→p, с→c, etc.
  • Grego (24 chars): α→a, β→b, ε→e, etc.
  • Matematico: Bold, Italic, Fraktur Unicode blocks
  • Fullwidth Latin: U+FF21..U+FF5A → ASCII
  • Superscript/Subscript: ᵃ→a, ᴬ→A
  • Preserva acentos portugueses: ç, ã, õ, á, é, í, ó, ú, â, ê, ô, à, ü, ñ

Geracao de variantes ofuscadas (para busca proativa):

  1. Substituicao de homoglifos (1 char por vez)
  2. Conversao completa para Braille
  3. Symbol-initial replacement
  4. Leet + sem espacos (para usernames)
  • Max 50 variantes por lista

Score Boosting ​

O BrandWatch aplica multiplicador de relevancia:

  • 1.5x se page_title contem o nome normalizado
  • Deduplicacao por URL normalizada
  • Mock mode ativo quando SERPAPI_API_KEY ausente
