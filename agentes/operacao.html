Guia de Operacao dos Agentes IA — Kubex ​

Guia tecnico de instalacao, operacao, monitoramento e manutencao dos 11 agentes KAAS.

Para a lista completa de agentes, veja agentes/index.md. Para prompts prontos, veja quick-start.md. Para diagramas de fluxo, veja fluxos.md.

1. Pre-requisitos ​

  • Node.js v18+ (recomendado v22+)
  • Docker Desktop rodando
  • Projeto Kubex em d:\Kubex
  • ANTHROPIC_API_KEY no .env (ou variavel de ambiente)
  • GitHub CLI (gh) para gestao de PRs

Verificar instalacao ​

bash
ls .aiox-core/personas/kubex/    # 11 personas
ls .aiox-core/skills/kubex/      # 8 skills
ls .aiox-core/workflows/kubex/   # 3 workflows
npx aiox-core doctor

2. Modelo de Operacao: Hibrido ​

8 agentes automaticos + 3 sob demanda. Detalhes de frequencia e custos em scheduler.md.

Tres formas de acionar agentes ​

FormaTokensDashboardQuando usar
Automatico (Scheduler)API (DeepSeek ~$0.01/ciclo)SimSRE, Docs, Director, Security, QA, DBA, Infra, Face Crawl
Dashboard ("Executar Agora")API (modelo do agente)SimQualquer agente sob demanda
Claude Code (prompt manual)Plano Max (gratis)NaoTrabalho interativo, sem custo

Rotina diaria do CEO (~2 min) ​

Manha:  Abrir dashboard /agentes → ver se SRE detectou algo overnight
18:00:  Ler relatorio do Director na timeline
Quando: Clicar em aprovacoes pendentes (se houver)

3. Monitoramento via Painel de Agentes ​

Acesse http://admin.kubex.test/agentes para monitorar agentes em tempo real.

Header Stats ​

Agentes Ativos: 11/11 | Custo Hoje: $2.50 | Aprovacoes Pendentes: 2 | Kill Switch: [Inativo]

Cards de Agentes ​

Grade com 11 cards, um por agente. Informacoes: status (idle/running/error), modelo, ultima execucao, proxima execucao, custo acumulado. Clique para expandir timeline e "Run Now".

Run Agent (On-Demand) ​

No card selecionado, digite instrucao customizada (ou deixe vazio para workflow padrao) e clique [Executar Agora].

Approval Queue ​

Decisoes sobre operacoes propostas por agentes. Risk levels: critical / high / medium / low. CEO decide: [Aprovar] ou [Rejeitar].

Mensagens entre Agentes ​

Comunicacao inter-agentes com TTL de 7 dias. Filtros: periodo (24h, 7d, todas), agente. Cores de expiracao: vermelho (< 24h), amarelo (< 72h), cinza (> 3d).

Terminal KAAS ​

Logs em tempo real via SSE. Levels: INFO / WARN / ERROR / DEBUG. Maximo 200 linhas (ring buffer).

4. Autoridade e Permissoes ​

Regras de Ouro ​

  1. Agentes NUNCA fazem merge em main sem aprovacao do CEO
  2. Agentes NUNCA editam .env, credentials, ou docker-compose.yml (prod)
  3. Agentes SEMPRE criam branches kaas/{agente}/{descricao}
  4. Operacoes destrutivas requerem aprovacao do CEO
  5. Kill switch so pode ser acionado pelo CEO

Escalonamento ​

Agente com problema → Director → CEO (Renato)

Comunicacao entre agentes (auto-trigger) ​

Agentes comunicam via Redis pub/sub. Quando um envia mensagem, o scheduler dispara o destinatario automaticamente.

Regras: cooldown 10min por agente, max 2 paralelos, agente precisa estar ativo e idle, mensagens expiram em 24h.

Fluxos comuns: SRE → Frontend/Backend/DBA, Security → Backend, QA → Backend, qualquer → Director.

6. Controles de Emergencia ​

bash
# Parar TODOS os containers
docker-compose down

# Restart de servico especifico
docker restart kubex-api-1

# Reset circuit breaker (via Redis)
docker exec kubex-redis-1 redis-cli DEL "kubex:kaas:cb:agent_sre"

# Kill switch global
curl -X POST http://admin.kubex.test/api/v1/admin/agents/kill-switch -d '{"active": true}'

# Sandbox toggle (habilitar/desabilitar escrita)
curl -X POST http://admin.kubex.test/api/v1/admin/agents/{slug}/sandbox-toggle

O kill switch ativa automaticamente se custo diario > KAAS_DAILY_BUDGET_USD ($50) ou > 10 falhas em 1 hora.

7. Gestao de Branches e PRs ​

bash
git branch -a | grep kaas/          # Ver branches de agentes
gh pr list --label kaas-bot          # PRs pendentes
gh pr review <N> --approve           # Aprovar
gh pr merge <N> --squash             # Mergear
git revert <COMMIT_HASH>            # Reverter mudanca

8. Economia de Tokens ​

Rules consolidadas ​

2 arquivos ao inves de 7 (~40% menos contexto):

  • .claude/rules/agent-operations.md
  • .claude/rules/compliance-review.md

Modelo por tarefa ​

TarefaModeloAgentes
Health check, docs, scan deps, DBADeepSeekSRE, Docs, Security, DBA
Testes, code review, fixes, frontend, infraSonnetQA, Backend, Infra, Frontend
Orquestracao, relatorios, priorizacaoOpusDirector

Dicas de economia ​

  • Agentes automaticos (SRE, Docs, Security) usam DeepSeek — custo minimo
  • Use Claude Code (plano Max) para trabalho interativo sem custo API
  • Relatorios do Director ficam salvos em docs/relatorios/ — nao precisa re-executar

9. Seguranca dos Agentes ​

PII Filtering (LGPD) ​

Todos os outputs de tools sao sanitizados antes de ir ao LLM. Dados filtrados: CPF, CNPJ, email, telefone, API keys, tokens hex, IPs internos, credenciais. Implementado em app/agents/pii_filter.py.

Sandbox mode ​

Sandbox ON (leitura)Sandbox OFF (escrita)
Arquivos, SELECT, Docker logs/ps, messaging+ escrita de arquivos, git checkout/add/commit/push

ON: SRE, DBA, Infra, Security, Director. OFF: Docs, QA, Backend, Frontend.

Rate limiting e Rollback ​

Cada agente tem max_calls_per_hour (Redis INCR). Antes de escrever um arquivo, salva conteudo original no Redis (TTL 7 dias).

10. Relatorios Automaticos ​

  • Diario (18:00): Director consulta acoes de todos os agentes, gera relatorio em docs/relatorios/diario-{data}.md
  • Semanal (segunda): Director executa weekly-meeting, salva em docs/relatorios/reuniao-{data}.md, cria aprovacoes para melhorias propostas
  • API: curl http://admin.kubex.test/api/v1/admin/agents/daily-report

11. Manutencao ​

  • Acoes antigas: retidas 90 dias. Limpar: POST /api/v1/admin/agents/cleanup-actions?retention_days=90
  • Aprovacoes expiradas: auto-rejeitadas apos 7 dias
  • Mensagens: TTL 7 dias no Redis, ultimas 5000 entradas
  • Budget: ajustar KAAS_DAILY_BUDGET_USD no .env ou por agente via API
