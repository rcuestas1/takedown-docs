Tabelas Principais ​

users ​

Tabela central. Autenticacao, perfil PF/PJ, endereco, estado de verificacao facial. Campos LGPD: cpf/cnpj em AES-256-GCM + cpf_hash/cnpj_hash em HMAC-SHA256 (busca O(1) sem decifrar).

audit_logs ​

Registro imutavel de auditoria. ondelete=SET NULL preserva logs apos exclusao de conta (LGPD).

Acoes registradas: login, login_failed, register, logout, 2fa_setup, 2fa_verify, 2fa_disable, password_change, password_reset, account_delete, profile_update, invite_create, takedown_create.

platform_settings ​

Singleton (CHECK id = 1). Configuracoes globais: maintenance_mode, registration_open, takedown_mode, storage_provider, search_engines (JSONB). Cacheado em Redis por 60s via get_cached_settings().

content_hashes ​

Hashes perceptuais (pHash + dHash + aHash) + thumbnail comprimido + CLIP embedding 512-d. LGPD: nunca armazena imagem original.

search_jobs ​

Jobs de busca (imagem ou nome). Status: pending → processing → completed|failed. public_id (hex 8 bytes) exposto na URL.

search_results ​

Resultados individuais. Cada URL encontrada gera 1 registro com score de similaridade (0.0-1.0).

search_purchases ​

Pay-per-search. Pagamento avulso via Asaas para desbloquear resultados de um job.

plans ​

Planos de assinatura configurados pelo admin. Campos: slug, price_cents, max_hashes, max_searches_month, max_takedowns_month, enabled_motors (JSON), scan_interval_hours, max_monitored_names.

subscriptions ​

Integracao Asaas. 1:1 com users. Status: pending → active → cancelled|overdue.

takedowns ​

DMCA ou Marco Civil. Status: pending → sent → still_online → verified_removed|escalated|judicial_recommended|failed.

takedown_verifications ​

Historico de verificacoes multi-metodo (http_status, content_analysis, phash_compare) com resultado e confianca.

creator_embeddings ​

Embeddings faciais criptografados (AES-256-GCM). Selfie deletada imediatamente apos gerar embedding (LGPD).

alerts ​

Alertas automaticos de monitoramento. Tipos: phash_match, face_match, name_match, telegram_match, discord_match, reddit_match, twitter_match, google_alert, clip_match, darkweb_match, instagram_match, video_match, judicial_recommended.

Migrations ​

Gerenciadas pelo Alembic com driver sync (psycopg2). Total: 65 revisoes em migrations/versions/.

Para comandos, veja Migrations.

Snapshot do Banco (Admin) ​

GET /admin/database/snapshot retorna estatisticas consolidadas. Modo estimado (default, ~10ms) via pg_stat_user_tables ou exato (?exact=true, ~500ms) via COUNT(*).

Pagina admin (/database) mostra: KPIs do banco, KPIs dos motores, registros por tabela, scans por motor, ultimos erros.
