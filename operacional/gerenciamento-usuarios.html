Gerenciamento de Usuarios ​

Guia operacional para administracao de usuarios do Kubex. Cobre desde a consulta de contas ate compliance LGPD, passando por gestao de planos, verificacao de identidade e controle de acesso. Todas as operacoes sao auditadas automaticamente.

Consultar e Buscar Usuarios ​

Acesse admin.kubex.test/users para o painel de gestao de usuarios.

O painel permite:

  • Busca por nome, email ou ID
  • Filtros por plano (Free, Basic, Pro, Enterprise), status (ativo, suspenso, pendente), e periodo de cadastro
  • Ordenacao por data de registro, ultimo login, ou quantidade de monitoramentos

Via API:

bash
# Listar usuarios com paginacao
curl -H "Cookie: access_token=<token>" \
  "https://admin.kubex.test/api/v1/admin/users?page=1&per_page=20"

# Buscar por email
curl -H "Cookie: access_token=<token>" \
  "https://admin.kubex.test/api/v1/admin/users?search=usuario@email.com"

# Detalhes completos de um usuario
curl -H "Cookie: access_token=<token>" \
  https://admin.kubex.test/api/v1/admin/users/<user_id>

Gerenciar Roles e Permissoes ​

O Kubex possui dois niveis de acesso: user (padrao) e admin.

bash
# Promover usuario a admin
curl -X PUT -H "Cookie: access_token=<token>" \
  -H "Content-Type: application/json" \
  -d '{"role": "admin"}' \
  https://admin.kubex.test/api/v1/admin/users/<user_id>

# Revogar acesso admin
curl -X PUT -H "Cookie: access_token=<token>" \
  -H "Content-Type: application/json" \
  -d '{"role": "user"}' \
  https://admin.kubex.test/api/v1/admin/users/<user_id>

DANGER

A promocao a admin concede acesso ao painel administrativo completo, incluindo dados de outros usuarios. Conceda apenas a operadores confiados.

Autenticacao de Dois Fatores (2FA) ​

O 2FA utiliza TOTP (Google Authenticator, Authy) e e obrigatorio para contas admin.

Habilitar 2FA para um usuario:

  1. Acesse o perfil do usuario em /users/<id>
  2. Na secao Seguranca, clique em Habilitar 2FA
  3. O usuario recebera instrucoes no proximo login

Forccar 2FA para todos os admins:

O 2FA e automaticamente obrigatorio para qualquer usuario com role admin. Ao promover um usuario, ele sera redirecionado para configurar 2FA no proximo acesso.

WARNING

Se um admin perder acesso ao app autenticador, o CEO pode resetar o 2FA via painel admin na secao Seguranca do perfil do usuario.

Para alterar o plano de um usuario:

bash
# Atualizar plano
curl -X PUT -H "Cookie: access_token=<token>" \
  -H "Content-Type: application/json" \
  -d '{"plan": "pro"}' \
  https://admin.kubex.test/api/v1/admin/users/<user_id>/subscription

# Consultar limites atuais do usuario
curl -H "Cookie: access_token=<token>" \
  https://admin.kubex.test/api/v1/admin/users/<user_id>/limits

O campo max_monitored_names respeita o limite do plano e e validado em tempo real. O cache de planos (plans_cache) garante fallbacks seguros caso o banco esteja indisponivel.

Verificacao de Identidade ​

Usuarios que solicitam o plano Enterprise passam por verificacao de identidade (selfie + documento).

Fluxo de revisao:

  1. Acesse admin.kubex.test/users?filter=pending_verification
  2. Clique no usuario para ver os documentos enviados
  3. Compare a selfie com o documento de identidade
  4. Aprove ou rejeite com justificativa
bash
# Aprovar verificacao
curl -X POST -H "Cookie: access_token=<token>" \
  -H "Content-Type: application/json" \
  -d '{"status": "approved"}' \
  https://admin.kubex.test/api/v1/admin/users/<user_id>/verification

# Rejeitar com motivo
curl -X POST -H "Cookie: access_token=<token>" \
  -H "Content-Type: application/json" \
  -d '{"status": "rejected", "reason": "Documento ilegivel"}' \
  https://admin.kubex.test/api/v1/admin/users/<user_id>/verification

TIP

Documentos de verificacao sao armazenados temporariamente e removidos automaticamente apos aprovacao/rejeicao, conforme politica LGPD.

Direitos LGPD ​

O Kubex implementa os direitos previstos na Lei Geral de Protecao de Dados.

Exportacao de Dados (Portabilidade) ​

bash
# Gerar pacote de exportacao para o usuario
curl -X POST -H "Cookie: access_token=<token>" \
  https://admin.kubex.test/api/v1/admin/users/<user_id>/data-export

O sistema gera um arquivo JSON com todos os dados pessoais do usuario. O link de download e enviado por email e expira em 48h.

Exclusao de Conta ​

bash
# Solicitar exclusao (soft delete + anonimizacao)
curl -X DELETE -H "Cookie: access_token=<token>" \
  https://admin.kubex.test/api/v1/admin/users/<user_id>

Comportamento da exclusao:

  • Dados pessoais (nome, email, telefone) sao anonimizados
  • Audit logs sao preservados com referencia por ID (sem PII), usando ondelete=SET NULL
  • Monitoramentos e resultados sao removidos via CASCADE
  • A exclusao e irreversivel

DANGER

A exclusao de conta e permanente. Confirme com o usuario antes de executar. O sistema solicita confirmacao dupla via painel admin.

API Keys para Usuarios B2B ​

Usuarios Enterprise podem gerar API keys para integracao programatica.

bash
# Listar API keys do usuario
curl -H "Cookie: access_token=<token>" \
  https://admin.kubex.test/api/v1/admin/users/<user_id>/api-keys

# Revogar uma API key
curl -X DELETE -H "Cookie: access_token=<token>" \
  https://admin.kubex.test/api/v1/admin/users/<user_id>/api-keys/<key_id>

Cada API key possui escopo definido (read-only ou read-write) e limite de requisicoes por minuto vinculado ao plano.

Sistema de Convites ​

Administradores podem convidar novos usuarios diretamente:

bash
# Enviar convite
curl -X POST -H "Cookie: access_token=<token>" \
  -H "Content-Type: application/json" \
  -d '{"email": "novo@empresa.com", "plan": "basic", "role": "user"}' \
  https://admin.kubex.test/api/v1/admin/invites

O convidado recebe um email com link de registro pre-aprovado. O convite expira em 7 dias. Todas as acoes de convite sao registradas no audit log.
