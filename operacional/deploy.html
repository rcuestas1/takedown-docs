Guia de Deploy em Producao ​

Tudo o que voce precisa para colocar o Kubex em producao pela primeira vez.

VPS: Hostinger • SSL: Cloudflare (gratis) • Dominio: kubex.com.br

1. Infraestrutura VPS ​

1.1 Requisitos minimos do servidor ​

RecursoMinimoRecomendado
CPU2 vCPUs4 vCPUs
RAM4 GB8 GB
Disco40 GB SSD80 GB SSD
OSUbuntu 22.04 LTSUbuntu 24.04 LTS
Docker24.x+27.x+
Docker Composev2.20+v2.30+

1.2 Setup inicial do servidor ​

bash
# 1. Atualizar sistema
sudo apt update && sudo apt upgrade -y

# 2. Instalar Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# 3. Instalar Docker Compose (ja incluso no Docker 24+)
docker compose version

# 4. Criar diretorios
sudo mkdir -p /opt/kubex
sudo mkdir -p /opt/kubex/backups/db
sudo chown -R $USER:$USER /opt/kubex

# 5. Clonar repositório
cd /opt/kubex
git clone https://github.com/SEU_USUARIO/kubex.git .

# 6. Configurar firewall
sudo ufw allow 22/tcp   # SSH
sudo ufw allow 80/tcp   # HTTP (Cloudflare)
sudo ufw allow 443/tcp  # HTTPS (Cloudflare)
sudo ufw enable

2. Cloudflare (SSL + CDN + Proteção) ​

2.1 Configuração DNS ​

TipoNomeConteúdoProxy
Akubex.com.brIP_DA_VPSProxied ☁️
Aadmin.kubex.com.brIP_DA_VPSProxied ☁️

2.2 SSL/TLS ​

  1. SSL Mode: Full (Strict)
  2. Gerar Origin Certificate:
    • Cloudflare Dashboard → SSL/TLS → Origin Server → Create Certificate
    • Salvar origin.pem e origin-key.pem em /opt/kubex/nginx/ssl/
    • Validade: 15 anos
  3. Always Use HTTPS: ON
  4. Minimum TLS Version: 1.2
  5. Automatic HTTPS Rewrites: ON

2.3 Segurança Cloudflare ​

  • WAF: Ativar regras gerenciadas (grátis)
  • Bot Fight Mode: ON
  • Challenge Passage: 30 minutos
  • Browser Integrity Check: ON
  • Under Attack Mode: Desligado (ativar se necessário)

2.4 Performance Cloudflare ​

  • Auto Minify: HTML, CSS, JS
  • Brotli: ON
  • Caching Level: Standard
  • Browser Cache TTL: 4 hours

4. Deploy ​

4.1 Primeiro deploy ​

bash
cd /opt/kubex

# 1. Configurar .env
cp .env.example .env
nano .env  # Preencher todas as variáveis

# 2. Copiar certificados SSL do Cloudflare
# Colocar origin.pem e origin-key.pem em nginx/ssl/

# 3. Build e start
docker compose -f docker-compose.yml -f docker-compose.prod.yml build
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

# 4. Rodar migrations (inclui Sprint 3: PIB, Facebook pages, whitelist, optout)
# Cadeia: b92c409d6179 → f4fabdf29959 → 818df16355dd → 0bc473bb7a72
docker compose exec api alembic upgrade head

# Verificar migrations aplicadas
docker compose exec api alembic current
docker compose exec api alembic history --verbose | head -20

# 5. Criar admin inicial
docker compose exec api python -c "
from app.database import async_session
from app.models.user import User
from app.core.security import hash_password
import asyncio

async def create_admin():
    async with async_session() as db:
        user = User(
            email='admin@kubex.com.br',
            hashed_password=await hash_password('TROCAR_SENHA_FORTE'),
            is_admin=True,
            is_active=True,
            is_verified=True,
            plan='premium',
        )
        db.add(user)
        await db.commit()
        print(f'Admin criado: {user.email}')

asyncio.run(create_admin())
"

# 6. Verificar health
curl https://kubex.com.br/api/health
curl https://kubex.com.br/api/ready

# 7. Verificar servicos Milvus (Face Recognition)
docker compose ps milvus etcd minio
# Esperado: milvus, etcd, minio — todos "Up" ou "healthy"
docker compose exec face-worker python -c "
from pymilvus import connections, utility
connections.connect(host='milvus', port=19530)
print('Milvus OK — colecoes:', utility.list_collections())
"

4.2 Deploy subsequente ​

bash
cd /opt/kubex
git pull origin main
docker compose -f docker-compose.yml -f docker-compose.prod.yml build
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
docker compose exec -T api alembic upgrade head

Configurar os seguintes secrets no GitHub (Settings > Secrets > Actions):

O workflow .github/workflows/deploy.yml executa automaticamente ao push na main.

5.1 Backup automatico (cron) ​

bash

# Adicionar:

bash

9. Teste Final pos-Deploy ​

Apos o deploy, execute estes testes para garantir que tudo esta funcionando:

bash
# 1. Health check
curl -s https://kubex.com.br/api/health | jq .

# 2. Testar login
curl -X POST https://kubex.com.br/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -H "Origin: https://kubex.com.br" \
  -d '{"email": "admin@kubex.com.br", "password": "..."}' \
  -c cookies.txt -v

# 3. Testar rate limiting (deve retornar 429 apos 5 tentativas)
for i in {1..7}; do
  curl -s -o /dev/null -w "%{http_code}\n" \
    -X POST https://kubex.com.br/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -H "Origin: https://kubex.com.br" \
    -d '{"email": "test@test.com", "password": "wrong"}'
done

# 4. Testar CSRF (request sem Origin deve ser rejeitado — esperado: 403)
curl -s -o /dev/null -w "%{http_code}\n" \
  -X POST https://kubex.com.br/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "test@test.com", "password": "test"}'

# 5. Verificar security headers
curl -sI https://kubex.com.br/ | grep -iE "x-frame|x-content|strict-transport|content-security"

# 6. Testar admin frontend
curl -s -o /dev/null -w "%{http_code}\n" https://admin.kubex.com.br/

# 7. Verificar Celery
docker compose -f docker-compose.prod.yml exec celery-worker celery -A app.tasks inspect ping

10. Rollback ​

Em caso de problemas apos deploy: ​

bash
# 1. Ver ultimo commit estavel
git log --oneline -5

# 2. Reverter para commit anterior
git checkout <commit-hash>

# 3. Rebuild
docker compose -f docker-compose.yml -f docker-compose.prod.yml build
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

# 4. Se migration falhou, reverter
docker compose exec api alembic downgrade -1

Em caso de emergencia: ​

bash
# Ativar modo de manutencao via API
curl -X PATCH https://admin.kubex.com.br/api/v1/admin/system/settings \
  -H "Cookie: access_token=<ADMIN_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"maintenance_mode": true, "maintenance_message": "Sistema em atualizacao"}'

# Ativar modo waitlist (pre-lancamento) via API
curl -X PATCH https://admin.kubex.com.br/api/v1/admin/system/settings \
  -H "Cookie: access_token=<ADMIN_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"waitlist_mode": true}'

# Nota: maintenance_mode tem prioridade sobre waitlist_mode.
# Ambos podem ser ativados/desativados pelo painel admin em /system.
