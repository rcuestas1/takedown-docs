Relatorios PDF ​

Geracao automatizada de relatorios profissionais em PDF para busca facial e monitoramento mensal.

Visao Geral ​

O Kubex gera relatorios PDF profissionais usando a biblioteca fpdf2. Os relatorios sao construidos pela classe KubexReportPDF, que estende FPDF com branding e formatacao padrao.

Arquivo principal: app/services/report_pdf_service.py

Endpoints ​

GET /v1/reports/face-search/{job_id}/pdf ​

Download do relatorio PDF de uma busca facial especifica.

Autenticacao: Cookie httpOnly (access_token) Acesso: Todos os planos (apenas jobs do proprio usuario)

http
GET /v1/reports/face-search/a1b2c3d4e5f67890/pdf
Cookie: access_token=...

Resposta: application/pdf (download automatico)

Filename: kubex-face-search-{job_id}.pdf

Conteudo do Relatorio ​

  1. Cabecalho Kubex — logo textual + data/hora de geracao
  2. Informacoes do Usuario — nome, email, data da busca, tempo de busca, tamanho do indice
  3. Resumo — total de matches, distribuicao por qualidade e categoria
  4. Matches Detalhados — tabela com ate 100 resultados:
    • Numero sequencial
    • Similaridade (%)
    • Qualidade (altissima/alta/media)
    • Dominio de origem
    • Categoria do site
    • URL (truncada)
  5. Aviso Legal — disclaimer sobre natureza dos resultados

GET /v1/reports/monitoring/monthly/pdf ​

Download do relatorio PDF de monitoramento mensal.

Autenticacao: Cookie httpOnly (access_token) Acesso: Todos os planos

http
GET /v1/reports/monitoring/monthly/pdf
Cookie: access_token=...

Resposta: application/pdf (download automatico)

Filename: kubex-monitoring-{YYYY-MM}.pdf

Conteudo do Relatorio ​

  1. Cabecalho Kubex — branding padrao
  2. Informacoes do Usuario — nome, email, periodo
  3. Estatisticas Gerais — total de buscas, alertas, takedowns no periodo
  4. Eventos de Monitoramento — tabela com ate 200 alertas dos ultimos 30 dias:
    • Data/hora
    • Motor responsavel
    • Plataforma
    • Descricao do evento
    • Severidade (info/warning/critical)
  5. Aviso Legal

Modelos de Dados ​

FaceSearchMatch ​

Representa um match facial para o relatorio:

python
@dataclass
class FaceSearchMatch:
    face_id: int
    similarity: float          # 0.0-1.0
    match_quality: str         # altissima/alta/media
    source_domain: str
    site_category: str
    source_url: str = ""
    page_title: str = ""

MonitoringEvent ​

Representa um evento de monitoramento:

python
@dataclass
class MonitoringEvent:
    date: str                  # "2026-03-22 14:30"
    motor: str                 # Nome do motor
    platform: str              # Plataforma de origem
    description: str           # Descricao do evento
    severity: str = "info"     # info | warning | critical

Geracao Programatica ​

Os relatorios podem ser gerados programaticamente:

python
from app.services.report_pdf_service import (
    FaceSearchMatch,
    generate_face_search_report,
    MonitoringEvent,
    generate_monitoring_report,
)

# Relatorio de busca facial
matches = [
    FaceSearchMatch(
        face_id=123,
        similarity=0.91,
        match_quality="altissima",
        source_domain="example.com",
        site_category="tube",
        source_url="https://example.com/page",
    ),
]
pdf_bytes = generate_face_search_report(
    user_name="Maria Silva",
    user_email="maria@example.com",
    matches=matches,
    search_time_ms=42,
    total_index_size=1000000,
)

# Relatorio de monitoramento
events = [
    MonitoringEvent(
        date="2026-03-22 14:30",
        motor="Reddit Monitor: Reddit",
        platform="reddit",
        description="Imagem encontrada em r/brasil",
        severity="warning",
    ),
]
pdf_bytes = generate_monitoring_report(
    user_name="Maria Silva",
    user_email="maria@example.com",
    events=events,
    total_searches=45,
    total_alerts=12,
    total_takedowns=3,
)
