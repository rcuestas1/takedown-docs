Pipeline de Bulk Crawling ​

Crawling em massa para populacao do KubexFaceIndex. Separado do crawling por-alvo (Crawler).

Visao Geral ​

O pipeline de bulk crawling e um sistema independente que percorre uma lista curada de dominios brasileiros para extrair e indexar faces no KubexFaceIndex. Opera na fila celery_low em background, sem impactar as buscas do usuario.

Seed List --> Crawl (Scrapling) --> Extrair imagens --> face_index.ingest_from_url --> Milvus

Diferenca do Crawler (Web Crawler):

AspectoCrawler (por-alvo)Bulk Crawling
TriggerBusca do usuarioCelery Beat (1h)
DominiosDefinidos pelo usuarioSeed list curada
ObjetivoEncontrar conteudo especificoPopular o indice facial
Queuecelery_defaultcelery_low
PrioridadeAltaBaixa (background)

Seed List BR ​

A seed list esta em app/engines/crawler/bulk_seed.py. Os dominios sao organizados por categoria, seguindo as particoes do Milvus:

Categorias ​

CategoriaDescricaoIntervaloExemplos
tubeSites de video adulto (alto volume de faces)24hxvideos, xnxx, pornhub
leakAgregadores de vazamento12hfapello
escortSites de acompanhantes BR48hphotoacompanhantes, garotacom
forumForums brasileiros72hgpguia
social_mediaRedes sociais (perfis publicos)48hflickr
imageboardImageboards24h(expansivel)
general_webWeb generica72h(expansivel)

Estrutura de um Seed ​

python
{
    "url": "https://www.xvideos.com",
    "category": "tube",
    "scrape_method": "basic",       # basic | dynamic | stealthy
    "crawl_paths": ["/new", "/best", "/profiles"],
    "interval_hours": 24,
}
CampoTipoDescricao
urlstrURL base do dominio
categorystrCategoria (particao Milvus)
scrape_methodstrMetodo de crawling: basic (httpx), dynamic (Playwright), stealthy (anti-bot)
crawl_pathslist[str]Caminhos a percorrer dentro do dominio
interval_hoursintIntervalo minimo entre crawls

Como Adicionar Novos Dominios ​

Editar a lista SEED_DOMAINS em app/engines/crawler/bulk_seed.py:

python
# app/engines/crawler/bulk_seed.py — adicionar ao SEED_DOMAINS
{
    "url": "https://novo-dominio.com.br",
    "category": "forum",            # deve ser uma particao valida
    "scrape_method": "dynamic",     # escolher conforme proteção do site
    "crawl_paths": ["/fotos", "/galeria"],
    "interval_hours": 48,
},

Regras:

  • Apenas dominios publicos com conteudo acessivel via web
  • robots.txt respeitado pelo crawler
  • User-Agent identificado como KubexBot
  • A categoria deve corresponder a uma particao do face_index.py

Redis Tracking de Last-Crawled ​

O sistema rastreia o timestamp da ultima vez que cada dominio foi crawleado usando chaves Redis:

kubex:bulk_seed:last:{domain}

Fluxo ​

  1. get_next_batch() le todos os seeds
  2. Para cada seed, verifica kubex:bulk_seed:last:{netloc} no Redis
  3. Se elapsed >= interval_hours * 3600 ou nunca crawleado: marca como "due"
  4. Ordena por staleness (mais atrasado primeiro)
  5. Retorna os top N (batch_size) dominios
python
# Exemplo de chave Redis
# Key: kubex:bulk_seed:last:www.xvideos.com
# Value: "1711324800.0"  (Unix timestamp)
# TTL: 7 dias (expira se dominio removido da seed list)

Apos crawl bem-sucedido, mark_crawled() atualiza o timestamp:

python
mark_crawled("https://www.xvideos.com", redis_url=settings.REDIS_URL)
# -> SET kubex:bulk_seed:last:www.xvideos.com "1711411200.0" EX 604800

Beat Schedule ​

O bulk crawling e acionado pelo Celery Beat a cada 1 hora:

python
# app/tasks/__init__.py — beat_schedule
"bulk-crawl-batch": {
    "task": "bulk_crawl.run_batch",
    "schedule": 3600.0,  # every 1 hour
    "options": {"queue": "celery_low"},
},

A task run_bulk_crawl_batch opera na queue celery_low para nao competir com tasks de busca do usuario.

Configuracao ​

Variaveis de Ambiente ​

env
BULK_CRAWL_ENABLED=true
BULK_CRAWL_BATCH_SIZE=10
BULK_CRAWL_MAX_IMAGES_PER_DOMAIN=200
VariavelDefaultDescricao
BULK_CRAWL_ENABLEDtrueHabilita/desabilita o pipeline de bulk crawling
BULK_CRAWL_BATCH_SIZE10Numero de dominios por execucao do beat
BULK_CRAWL_MAX_IMAGES_PER_DOMAIN200Maximo de imagens extraidas por dominio/crawl

Global Cap ​

Cada execucao do batch tem um cap de 500 tasks despachadas para a fila celery_face. Isso previne inundacao da fila quando muitos dominios estao atrasados simultaneamente.

python
MAX_TASKS_PER_BATCH = 500  # Global cap to prevent celery_face queue flood

Fluxo Completo ​

[Celery Beat - 1h]
        |
        v
run_bulk_crawl_batch()
        |
        v
BULK_CRAWL_ENABLED? ── false ──> skip
        |
      true
        v
get_next_batch(limit=10)  ── 0 domains ──> idle
        |
     N domains
        v
Para cada dominio:
  ├── ScraplingCrawlerService.fetch_html(url, method)
  ├── extract_image_urls(html)
  ├── Limitar a MAX_IMAGES_PER_DOMAIN
  ├── Para cada imagem:
  │     └── face_index.ingest_from_url.delay(img_url, category)
  └── mark_crawled(url)
        |
        v
Return: {status, domains, images, faces_dispatched}

Monitoramento ​

Verificar o status do bulk crawling:

bash
# Logs do worker
docker compose logs worker --tail 50 | grep bulk_crawl

# Verificar fila
docker compose exec worker celery -A app.tasks inspect active_queues

# Verificar dominios no Redis
docker compose exec redis redis-cli KEYS "kubex:bulk_seed:last:*"

# Verificar ultimo crawl de um dominio
docker compose exec redis redis-cli GET "kubex:bulk_seed:last:www.xvideos.com"
