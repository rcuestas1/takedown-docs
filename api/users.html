Usuarios — Endpoints e Regras de Negocio ​

Referencia completa: Swagger UI — Users

Todos os endpoints requerem autenticacao via cookie httpOnly (access_token).

Retorna perfil completo. CPF e CNPJ sao decriptados e retornados mascarados (ex: ***.123.456-**).

PATCH /users/me ​

Atualiza perfil. Envia apenas os campos que deseja alterar (exclude_unset).

Campos editaveis ​

CampoValidacao
full_nameString
person_typeindividual ou business
phoneTelefone BR valido
avatar_urlURL
company_nameString
birth_dateData passada, idade minima 16 anos
razao_socialString (PJ)
nome_fantasiaString (PJ)
cpfCPF valido (encriptado AES-256, hash HMAC para unicidade)
cnpjCNPJ valido (encriptado AES-256, hash HMAC para unicidade)
cepCEP valido
street, neighborhood, address_number, address_complement, city, state, countryStrings de endereco
notification_preferencesDict JSON

Seguranca CPF/CNPJ: Valores sao validados, encriptados (AES-256-GCM) e armazenados com hash HMAC para garantir unicidade sem expor texto claro. Se CPF/CNPJ ja estiver cadastrado por outro usuario, retorna 409 Conflict.

Gera audit log update_profile.

PATCH /users/me/password ​

Altera senha. Exige senha atual para confirmacao.

Request ​

json
{
  "current_password": "senhaAtual123",
  "new_password": "novaSenha456"
}

Regras ​

  • Senha atual deve ser correta (401 se incorreta)
  • Nova senha deve ser diferente da atual (400 se igual)
  • Hash bcrypt via asyncio.to_thread
  • Gera audit log change_password

POST /users/me/invites ​

Gera codigo de convite (6 letras maiusculas, ex: ABC DEF).

Regras ​

  • Limite configuravel via PlatformSettings.max_invites_per_user
  • Convite expira apos invite_bonus_days dias
  • Gera audit log create_invite

Response ​

json
{
  "code": "ABCDEF",
  "expires_at": "2026-04-30T00:00:00"
}

GET /users/me/invites ​

Lista convites gerados pelo usuario com paginacao.

Query Params ​

ParamDefaultDescricao
page1Pagina
per_page10Itens por pagina (max 50)

GET /users/me/data-summary (LGPD Art. 18, II) ​

Retorna contagens de cada tipo de dado coletado do usuario, sem expor os dados em si.

Response ​

json
{
  "content_hashes": 12,
  "search_jobs": 45,
  "alerts": 8,
  "takedowns": 3,
  "monitored_names": 2,
  "audit_logs": 156,
  "face_search_consents": 1
}

GET /users/me/export (LGPD Art. 18, V) ​

Portabilidade de dados pessoais. Retorna todos os dados do usuario em JSON, incluindo:

  • Perfil completo (com CPF/CNPJ decriptados)
  • Content hashes registrados
  • Search jobs e resultados
  • Alertas recebidos
  • Takedowns realizados
  • Nomes monitorados
  • Audit logs do usuario

Rate limit: 3 req/min. Gera audit log export_data.

DELETE /users/me/delete (LGPD Art. 18) ​

Exclui conta e todos os dados do usuario via CASCADE DELETE:

  1. Subscriptions, alerts, searches, content_hashes, takedowns, monitored_names
  2. Embeddings faciais removidos do Milvus via delete_by_user()
  3. Chaves Redis limpas
  4. Audit logs preservados anonimizados (user_id = NULL via SET NULL)
  5. Cookies de sessao removidos

Gera audit log delete_account (anonimizado).
