Activity & Audit Logs ​

Activity Logs dos Motores ​

Admin page: /activity-logsArquivo: app/api/v1/admin/activity_logs_admin.py

Os Activity Logs registram toda a atividade dos motores de busca em tempo real. Cada execucao gera um trace_id unico que conecta os logs da API e do Celery, permitindo rastrear o fluxo completo de uma operacao.

Endpoints ​

MetodoRotaDescricao
GET/admin/activity-logsLista paginada com filtros (motor_id, level, trace_id, task_id, search, date range)
GET/admin/activity-logs/streamSSE (Server-Sent Events) em tempo real via Redis Pub/Sub (ACTIVITY_CHANNEL)
GET/admin/activity-logs/traces/{trace_id}Timeline completa de uma execucao
GET/admin/activity-logs/statsResumo das ultimas 24h por level e motor

Schema: EngineActivityLog ​

SSE Streaming ​

O endpoint /stream permite que o frontend receba logs em tempo real sem polling.

  • Retorna StreamingResponse com content-type: text/event-stream
  • Dados publicados via Redis Pub/Sub no canal ACTIVITY_CHANNEL
  • Frontend consome via EventSource API nativa do browser
  • Nginx prod: proxy_buffering off e timeout 3600s configurados para este endpoint
┌──────────┐    Pub/Sub     ┌──────────┐     SSE      ┌──────────┐
│  Celery  │ ──────────────>│  Redis   │ ────────────>│ Frontend │
│  Worker  │  ACTIVITY_CH   │  Server  │  /stream     │  Admin   │
└──────────┘                └──────────┘              └──────────┘

Audit Logs ​

Admin page: /audit-logsArquivo: app/api/v1/admin/audit_logs.py

Os Audit Logs registram todas as acoes sensiveis do sistema para fins de seguranca e conformidade com a LGPD.

Endpoints ​

MetodoRotaDescricao
GET/admin/audit-logsLista paginada com filtros (action, user_id, ip, date_from, date_to). LEFT JOIN com User para mostrar email. Window function para total count (single query).
GET/admin/audit-logs/export/csvExportacao CSV com StreamingResponse

Acoes registradas ​

As seguintes acoes sao capturadas automaticamente no audit log:

  • login — Login de usuario
  • register — Registro de nova conta
  • logout — Logout
  • setup_2fa — Ativacao de autenticacao em dois fatores
  • disable_2fa — Desativacao de 2FA
  • change_password — Alteracao de senha
  • delete_account — Exclusao de conta
  • approve_identity — Aprovacao de identidade (admin)
  • reject_identity — Rejeicao de identidade (admin)
  • revoke_embedding — Revogacao de embedding facial
  • Demais operacoes administrativas

LGPD ​

Os audit logs sao preservados de forma anonimizada quando um usuario exclui sua conta. O relacionamento com a tabela User utiliza ondelete=SET NULL, garantindo que os registros de auditoria permanecem no sistema sem dados pessoais identificaveis, em conformidade com a Lei Geral de Protecao de Dados.
