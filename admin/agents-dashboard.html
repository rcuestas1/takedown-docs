Painel de Agentes IA — Dashboard KAAS ​

Gerenciamento centralizado dos 10 agentes IA que operam o Kubex, com monitoramento em tempo real, mensagens inter-agentes, logs de execucao e controles operacionais.

Acesso: http://admin.kubex.test/agentes (dev) | https://admin.kubex.com.br/agentes (prod)

Para descricao detalhada de cada agente (papeis, modelos, frequencias), ver KAAS Documentacao.

Visao Geral ​

+----------------------------------------------------------+
|  Painel de Agentes IA (KAAS)                             |
|----------------------------------------------------------|
|  [Header Stats] Ativos: 10/10 | Custo: $2.50 | Pendentes: 2 | Kill Switch |
|  [Agent Cards Grid] 10 cards com status, modelo, custo   |
|  [Run Agent + Timeline] Instrucao customizada + acoes     |
|  [Approval Queue] Aprovacoes pendentes (CEO)              |
|  [Custos por Agente] Tabela: hoje, mes, restante, calls   |
|  [Mensagens] Filtros: periodo, agente, expandir, TTL      |
|  [Terminal KAAS] Logs SSE em tempo real                    |
+----------------------------------------------------------+

1. Header Stats ​

MetricaDescricaoAlerta
Agentes AtivosQuantos dos 10 agentes estao is_active=true> 2 inativos: investigar
Custo HojeSoma de cost_today_usd> $30: pausar agentes nao-criticos
Aprovacoes PendentesCount de AgentApproval com decision=NULL> 0: requer acao do CEO
Kill SwitchEmergencia: kubex:kaas:kill_switchAtivo = todos agentes param

Kill Switch: Pausa todos os agentes imediatamente. KAAS Scheduler nao dispara novas execucoes. Agentes nao respondem a mensagens (exceto CEO override).

2. Agent Cards ​

Grade de 10 cards. Cada card exibe: toggle ativo/inativo, descricao do papel, status badge, modelo, ultima execucao, proxima execucao, horario de operacao, custo hoje, falhas consecutivas.

Status Badge ​

StatusSignificado
idleAguardando proximo ciclo
runningExecutando agora
errorUltima execucao falhou
pausedDesativado manualmente
disabledDesabilitado pelo kill switch

3. Run Agent + Timeline ​

Executar Agora ​

  1. Selecionar agente no grid
  2. Digitar instrucao customizada (opcional — em branco executa workflow padrao)
  3. Clicar [Executar Agora]
  4. Agente inicia em background (pool de workers)
  5. Timeline atualiza em tempo real

Timeline ​

Ultimas 20 acoes do agente selecionado. Colunas: timestamp (HH:MM), status badge (success/running/error), descricao, duracao, modelo. Clicar numa acao expande o relatorio completo.

4. Approval Queue ​

Decisoes do CEO sobre operacoes propostas pelos agentes. Cada item exibe: nivel de risco, descricao, contexto, botoes [Aprovar] e [Rejeitar].

Niveis de risco:

NivelCorDescricao
criticalVermelhoOperacao destrutiva
highLaranjaImpacto significativo
mediumAmareloImpacto moderado
lowVerdeImpacto minimo

5. Custos por Agente ​

Tabela com colunas: Agente, Hoje ($), Mes ($), Restante ($), Calls.

  • Hoje — kubex:kaas:cost:{agente}:{YYYY-MM-DD}
  • Mes — Custo acumulado do mes corrente
  • Restante — daily_budget_usd - cost_today (negativo = excedido)
  • Calls — Numero de execucoes hoje

6. Mensagens entre Agentes ​

Filtros ​

FiltroOpcoes
PeriodoUltimas 24h, Ultimos 7 dias, Todas
AgenteTodos, SRE, Director, Docs, Backend, etc.

Tipos de Mensagem ​

TipoBadgeDescricao
task_request[Tarefa]Um agente pedindo outro fazer algo
status_update[Status]Relatorio de progresso
escalation[Escalacao]Problema que precisa de CEO
conflict[Conflito]Dois agentes querem mesmo recurso

Prioridade ​

critical (vermelho), high (laranja), normal (azul), low (cinza).

TTL ​

Mensagens expiram apos 7 dias (168 horas). Indicador de tempo restante: vermelho (< 24h), amarelo (< 72h), cinza (> 3 dias).

Clique em qualquer mensagem para expandir texto completo com formatacao Markdown.

7. Terminal KAAS ​

Logs em tempo real via SSE (Server-Sent Events). Levels: INFO (verde), WARN (amarelo), ERROR (vermelho), DEBUG (cinza, dev only). Auto-scroll ativo.

8. API dos Agentes ​

GET /v1/admin/agents/ ​

Retorna lista de agentes com: id, slug, name, persona_name, role, model, fallback_model, frequency_minutes, schedule_start/end, is_active, sandbox_mode, daily_budget_usd, status, last_run_at, next_run_at, total_cost_usd, consecutive_failures. Inclui kill_switch_active e total_cost_today_usd.

GET /v1/admin/agents/{slug}/actions ​

Retorna ultimas acoes do agente: id, action_type, description, status, model_used, cost_input_tokens, cost_output_tokens, duration_ms, created_at.

GET /v1/admin/agents/messages ​

Query Params:

ParamTipoDefaultDescricao
limit1-20050Quantidade de mensagens
hours1-168-Filtrar pelos ultimos N horas
prioritystr-low, normal, high, critical
msg_typestr-task_request, status_update, escalation, conflict

TTL: mensagens expiram apos 7 dias (168 horas).

9. Troubleshooting ​

ProblemaSolucao
Painel nao carregaVerificar autenticacao admin, testar GET /api/v1/admin/agents/
Mensagens nao aparecemVerificar is_active=true, Redis keys kubex:kaas:*, aguardar SSE (ate 5s)
Timeline vazioAgente ainda nao executado ou historico limpo (retencao: 90 dias)
Kill Switch nao respondeVerificar Redis: get kubex:kaas:kill_switch, reiniciar Redis
